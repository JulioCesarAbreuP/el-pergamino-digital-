Lab 26 – Evaluación de Hardenización de Sistemas Linux

Aplicación de técnicas de hardening para reforzar la seguridad en entornos Linux

Objetivo

Evaluar el nivel de endurecimiento (hardening) de un sistema Linux mediante la revisión de configuraciones críticas, servicios activos, permisos, políticas de autenticación y controles de auditoría, con el fin de reducir la superficie de ataque.

Alcance y limitaciones

El laboratorio se ejecuta exclusivamente en entornos controlados. No cubre auditorías forenses ni respuesta a incidentes. No debe aplicarse directamente en sistemas de producción sin autorización.

Requisitos previos

  • Conocimientos básicos de administración Linux
  • Acceso root o sudo
  • Máquina Linux (Ubuntu, Debian, Rocky o similar)

Arquitectura del entorno

Una máquina Linux objetivo evaluada desde la propia consola local o vía SSH en una red virtual aislada.

Herramientas y entorno técnico

  • Lynis
  • chkrootkit
  • auditd
  • systemctl

Procedimiento técnico

sudo apt install lynis
sudo lynis audit system
systemctl list-unit-files --type=service
sudo chkrootkit

Resultados esperados

  • Listado de debilidades de configuración
  • Servicios innecesarios identificados
  • Puntuación de hardening del sistema

Análisis de resultados

Los hallazgos permiten identificar configuraciones inseguras, servicios expuestos y prácticas débiles que incrementan el riesgo de compromiso del sistema.

Medidas

  • Deshabilitar servicios innecesarios
  • Aplicar políticas de contraseñas fuertes
  • Habilitar auditoría y logging

Contramedidas

  • Hardening continuo
  • Auditorías periódicas
  • Automatización con benchmarks CIS

Riesgos y advertencias

Un hardening mal aplicado puede afectar la disponibilidad del sistema. Validar siempre en entornos de prueba.

Evidencias

Espacio reservado para capturas de pantalla, logs y reportes.

Validación cruzada

  • CIS Linux Benchmarks
  • MITRE ATT&CK
  • NIST SP 800

Certificaciones asociadas

Security+, Linux+, CySA+, GSEC

Analogía técnica

Como reforzar murallas, cerrar portones y limitar accesos en un castillo medieval.

Nivel de dificultad

Intermedio – Avanzado

Checklist final

  • ✔ Auditoría ejecutada
  • ✔ Hallazgos documentados
  • ✔ Medidas aplicadas

Reflexión estratégica

El hardening no es un estado final, sino un proceso continuo que debe adaptarse a nuevas amenazas y cambios operativos.
