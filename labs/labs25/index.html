Lab 25 – Análisis de Seguridad de APIs REST y Técnicas de Autenticación

Exploración de mecanismos de autenticación y control de acceso en APIs REST

Objetivo

Este laboratorio capacita al analista en identificar y evaluar mecanismos de seguridad en APIs REST, validando autenticación, autorización, control de acceso y resistencia a ataques comunes como fuerza bruta, manipulación de tokens y bypass de controles.

Alcance y limitaciones

Pruebas exclusivamente en entornos controlados. No se aplican técnicas fuera de redes propias autorizadas.

Requisitos previos

  • Conocimientos de HTTP/REST
  • Herramientas como Postman, curl, Burp Suite
  • API Vulnerable de prueba (local o en laboratorio)

Entorno de laboratorio

API REST con autenticación básica, token JWT y OAuth2 para pruebas de autenticación y autorización.

Desarrollo

Enviar peticiones con herramientas como curl o Postman para validar autenticación (200/401/403) y manipular tokens para analizar bypass.

Resultados esperados

  • Detección de controles de autenticación
  • Evaluación de decisiones de autorización
  • Comprobación de resistencia frente a modificación de tokens

Análisis de resultados

Explicar cómo los controles HTTP 401/403 y JWT invalidados impiden o permiten accesos no autorizados.

Medidas

  • Documentar fallas observadas en autenticación
  • Revisar lógica de control de sesiones

Contramedidas

  • Implementar validación fuerte de tokens
  • Aplicar límites de intentos y bloqueo de fuerza bruta

Riesgos y advertencias

La manipulación de tokens puede provocar accesos no autorizados si se aplica fuera de entornos controlados. Solo uso educativo.

Evidencias

Espacio para capturas de pantalla de peticiones, respuestas y logs.

Validación cruzada

  • MITRE ATT&CK API Abuse
  • CIS Controls – Application Software Security
  • NIST SP 800-95 – Guide to Secure Web Services

Certificaciones asociadas

Security+, CEH, eWPTx2

Glosario asociado

API, JWT, OAuth2, Autenticación, Autorización

Preguntas de refuerzo

¿Cómo validar JWT? ¿Qué diferencia hay entre autenticación y autorización?

Rúbrica de evaluación

  • ✔ Autenticación comprobada
  • ✔ Autorización verificada
  • ✔ Tokens evaluados

Recursos complementarios

  • OWASP API Security
  • OAuth 2.0 RFC
