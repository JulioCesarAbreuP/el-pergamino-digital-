Lab 25 – Análisis de Seguridad de APIs REST

Evaluación de autenticación, autorización y control de acceso en servicios REST

Objetivo

Analizar los mecanismos de seguridad implementados en APIs REST, identificando debilidades en autenticación, autorización y manejo de tokens.

Alcance y limitaciones

Las pruebas se realizan únicamente en entornos controlados y APIs de laboratorio.

Requisitos previos

  • Conocimientos básicos de HTTP y REST
  • Uso de herramientas como curl o Postman
  • API vulnerable de prueba

Entorno de laboratorio

API REST con autenticación basada en tokens (JWT u OAuth2), desplegada en red virtual controlada.

Desarrollo

Se envían solicitudes autenticadas y no autenticadas para evaluar respuestas 200, 401 y 403, además de manipular tokens para comprobar controles.

Resultados esperados

  • Identificación de controles de autenticación
  • Validación de autorización por rol
  • Detección de fallos en validación de tokens

Análisis de resultados

Se interpretan las respuestas HTTP y el comportamiento del backend ante solicitudes manipuladas.

Contramedidas

  • Validación estricta de tokens
  • Control de acceso por rol
  • Registro y monitoreo de eventos

Riesgos y advertencias

El análisis de APIs fuera de entornos autorizados puede constituir una violación legal.

Evidencias

Espacio reservado para capturas de peticiones, respuestas y logs.

Mapa de integración

Continúa el enfoque de seguridad en aplicaciones web y servicios.
